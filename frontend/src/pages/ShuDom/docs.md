## Техническое задание (ТЗ): Разработка игры ShuDom: Techno Mania

> **Цель проекта:** Разработка кроссплатформенной Match-3 игры с глубокой интеграцией в существующую e-commerce инфраструктуру Technodom, ориентированной на повышение retention, времени сессии и конверсии в покупку.

### 1. Архитектура и Технологический стек

Проект требует отказоустойчивой микросервисной архитектуры, способной выдержать резкие скачки трафика (особенно в моменты push-рассылок и национального релиза).

| Компонент | Технология / Требование |
| --- | --- |
| **Платформы** | Native iOS, Native Android, Web-версия (на сайте technodom.kz). |
| **Frontend (Игра)** | Unity (интеграция в Native через Unity as a Library) **ИЛИ** Phaser.js (для легковесного Web). Обязательна глубокая Touch-оптимизация. |
| **Backend** | Node.js + Express.js. Микросервисная архитектура для разделения игровой логики и API интеграций. |
| **Базы данных (Основная)** | PostgreSQL (хранение профилей, инвентаря, логов транзакций, прогресса комнат). |
| **Базы данных (Кэш)** | Redis (кэширование сессий, лидербордов реального времени, статуса энергии). |
| **Уведомления** | Firebase Cloud Messaging (FCM) для персонализированных push-уведомлений. |

---

### 2. Интеграция с внешней экосистемой (API)

Бэкенд игры не является изолированным. Требуется разработка шлюза для двустороннего обмена данными с текущей инфраструктурой Technodom:

* **API Каталога Technodom:** Синхронизация реальных товаров в игре (цены, наличие, характеристики). Переход из игры прямо в карточку товара по кнопке «Купить / Купить в рассрочку 0-0-12».
* **API CRM-системы:** Автоматическая генерация, валидация и привязка к пользователю униканых промокодов (скидки, бесплатная доставка).
* **API Системы Лояльности:** Начисление и списание кэшбэка, интеграция с бонусными счетами, конвертация реальных покупок пользователя (по чекам) во внутриигровую энергию и билеты.
* **API Платежных шлюзов:** Интеграция с Kaspi и Halyk Bank для прямой обработки оплат при переходе из игры (или для будущей покупки внутрииговой валюты).

---

### 3. Безопасность и Античит (High Priority)

Учитывая призовой фонд в 200 млн тенге, безопасность архитектуры является критическим фактором. Вся логика, влияющая на баланс и шансы, должна находиться строго на сервере.

* **Серверная авторитативность:** Клиент (приложение) только отправляет действия игрока и визуализирует результат. Бэкенд валидирует **каждый ход**, пересчитывает очки и проверяет возможность комбо.
* **Защита аккаунтов:** Жесткий лимит на создание профилей (по Device ID / номеру телефона).
* **Эмиссия билетов (Блокчейн):** Золотые билеты для гранд-финала генерируются с уникальным ID, жестко привязываются к аккаунту (передача запрещена) и дублируются в блокчейн-реестр для обеспечения 100% прозрачности и неизменяемости перед розыгрышем.
* **Система анализа аномалий:** Автоматические триггеры на подозрительное поведение. Примеры: прохождение уровня за невозможные 2 секунды, подмена пакетов, начисление аномального количества очков. Наказание — теневой бан или блокировка на участие в розыгрышах.
* **Админ-панель (Модерация):** Инструмент для ручной выгрузки логов сессий пользователей из Топ-100 перед выдачей еженедельных призов.

### 4. Игровая логика (Core & Meta Loop)

Разработчикам клиентской части необходимо реализовать следующие механики:

* **Грид и механика:** Игровое поле 8x8. Лимит ходов (20-30 на уровень).
* **Алгоритм генерации поля:** Защита от «тупиковых» генераций (всегда должен быть доступен хотя бы один ход).
* **Система энергии:** 5 слотов. 1 сгорает при проигрыше. Таймер восстановления — 30 минут (расчет времени должен идти на сервере, чтобы избежать накрутки через изменение времени на устройстве).
* **Прогрессия Умного Дома:** Сохранение состояния 10 виртуальных комнат (25 уровней на комнату). Открытие 3D-моделей техники каждые 5 уровней.

---

### 5. Аналитика и A/B тестирование

Для управления жизненным циклом продукта (LiveOps) требуется внедрение систем трекинга:

* **Событийная аналитика:** Интеграция Firebase Analytics / Google Analytics для отслеживания кастомных ивентов (начало уровня, проигрыш, использование бустера, клик по карточке реального товара).
* **Внутренний BI-дашборд:** Визуализация метрик DAU, MAU, Retention 1/7/30, конверсии в покупку реальных товаров из игры.
* **Система A/B тестирования:** Инфраструктура для динамического изменения конфигов с сервера (балансировка сложности конкретных уровней, изменение шансов выпадения призов, корректировка цен на бустеры без обновления самого приложения).

### 6. План развертывания (Roadmap)

| Этап | Длительность | Ключевые задачи |
| --- | --- | --- |
| **Этап 1: MVP и Core-механики** | 2 месяца | Создание базового геймплея Match-3, разработка API бэкенда, интеграция с авторизацией Technodom, настройка PostgreSQL/Redis. |
| **Этап 2: Soft Launch** | 2-3 недели | Релиз только для Алматы и Астаны (лимит ~10 000 юзеров). Нагрузочное тестирование, сбор логов античита, балансировка сложности уровней 1-30. |
| **Этап 3: Национальный запуск** | 1 неделя | Масштабирование серверов, релиз по всему Казахстану, старт выдачи Золотых билетов, включение записи в блокчейн. |
| **Этап 4: Поддержка (LiveOps)** | Постоянно | Добавление новых уровней каждые 2 недели. Разработка архитектуры для "Сезонных эвентов" (замена текстур и наград без переписывания ядра). |
| **Этап 5: Монетизация (Опционально)** | Будущие релизы | Подключение биллинга для покупки энергии (99 тг/5 зарядов), реализация "Премиум-пропуска" и кастомной косметики. |
