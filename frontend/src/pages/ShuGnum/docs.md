## Техническое задание (ТЗ): Разработка WebView мини-игры «ShuGnum: Оливье Doda»

> **Цель проекта:** Создание легковесного (до 8 МБ) аркадного 2D-слэшера для интеграции в нативные приложения Magnum Go (iOS/Android) с целью увеличения времени сессии, DAU и конверсии в использование промокодов.
> **Форм-фактор:** HTML5 WebView-модуль, бесшовно связанный с ядром основного приложения и программой лояльности Magnum Club.

### 1. Архитектура и Технологический стек

Игра должна мгновенно загружаться поверх основного интерфейса магазина и не вызывать утечек памяти на слабых устройствах.

| Компонент | Технологии / Требования |
| --- | --- |
| **Frontend (Игровой клиент)** | Phaser.js или PixiJS (HTML5 Canvas / WebGL). Использование спрайтовых атласов (Sprite Sheets) для сжатия 3D-моделей в 2D-графику. Строгий лимит веса: **5-8 МБ**. |
| **Взаимодействие с Native** | JavaScript Bridge (JSBridge) для передачи токенов авторизации пользователя из приложения Magnum в WebView и возврата команд на закрытие окна. |
| **Backend (Core API)** | Node.js. Микросервис, изолированный от основной инфраструктуры магазина, чтобы пиковые игровые нагрузки не "положили" корзину покупок. |
| **Базы данных** | PostgreSQL (хранение прогресса "Тазика Оливье", баланса бонусов, логов). |
| **Кэширование** | Redis (для обработки еженедельных лидербордов в реальном времени и синхронизации PvP-дуэлей). |

---

### 2. Игровая механика и Математика (Frontend Core)

Разработчикам клиента нужно реализовать систему спавна объектов с учетом физики полета (параболическая траектория) и систему распознавания свайпов (Raycasting).

* **Режимы таймера:** Блиц (30 сек), Стандарт (60 сек), Продвинутый (90 сек).
* **Система начисления очков (Scoring):**
* Базовый продукт: +10 очков.
* Редкий продукт (казы, икра): +50 очков.
* Идеальная нарезка (по центру хитбокса): +5 бонусных очков.
* Комбо (пересечение 3+ хитбоксов одним непрерывным свайпом): Множитель x2 к сумме очков разрезанных объектов.


* **Система штрафов (Penalties):**
* Камень: -20 очков и штраф таймера -5 секунд.
* Испорченный продукт: -15 очков.
* "Дырявый носок": -10 очков + триггер визуального эффекта (замутнение экрана на 2-3 секунды).


* **Система баффов (Power-Ups):** Реализация вероятностного дропа (RNG) при спавне объектов:
* Скутер (Шанс 15%): Изменение глобального масштаба времени (Time Scale) до 0.5 на 10 сек.
* Карта Magnum (Шанс 10%): Глобальный множитель очков x2 на 10 сек.
* Магнит (Шанс 8%): Скрипт принудительного стягивания всех активных объектов (кроме штрафных) в центр экрана с авто-разрезом.
* Шеф-нож (Шанс 12%): Увеличение радиуса Raycast-линии свайпа в 2 раза на 10 сек.

### 3. Интеграция с Magnum Club (API & Мета-игра)

Бэкенд игры должен быть интегрирован с процессингом программы лояльности:

* **Прогрессия "Тазик Оливье":** 1 разрезанный продукт = 100 грамм. Цель — 10 кг (100 продуктов). При достижении порога бэкенд вызывает API Magnum Club для начисления 1000 бонусов и выдачи бейджа.
* **Мгновенные призы (Instawins):** При разрезании редкого продукта отправляется запрос на сервер. Сервер по таблице вероятностей (Loot table) возвращает результат: пустой ответ, промокод (5-10%), бонусы (100-500) или купон на доставку.
* **PvP "Вызов друга":** Генерация уникального URL (Deep link). При переходе второго игрока сервер фиксирует его результат и сравнивает с результатом создателя ссылки, автоматически начисляя 1000 бонусов победителю.
* **Социальные действия:** API-эндпоинты для обработки успешного шеринга (Share API) с начислением +300 бонусов и реферальной установки (+500 бонусов).

### 4. Безопасность и Античит (High Priority)

Поскольку игра напрямую выдает скидки и материальные ценности (iPhone 15 Pro, PS5), клиентская часть считается скомпрометированной по умолчанию.

* **Серверная валидация сессии:** Клиент отправляет массив (лог) разрезанных объектов с таймстампами. Сервер проверяет:
* Могло ли сгенерироваться такое количество редких продуктов за 60 секунд?
* Физически возможна ли такая скорость нарезки (защита от авто-кликеров/макросов)?
* Реалистично ли количество комбо?


* **Блокировка (Shadowban):** При выявлении аномалий (например, 10000 очков за 30 секунд), сессия не засчитывается в лидерборд, аккаунт помечается флагом для проверки.
* **Ручная модерация:** Админ-панель для проверки логов сессий топ-100 игроков недели перед выдачей главных призов.

---

### 5. Сезонность и LiveOps (Динамические конфиги)

Чтобы менять визуальное оформление (Оливье, Баурсак, Беш-Баттл, Окрошка) без релиза обновлений в App Store / Google Play:

* **JSON-конфигурация:** При запуске игра запрашивает с сервера конфиг-файл, который определяет: набор текстур (огурцы или тесто), фон (кухня или юрта), звуковой пак, активные штрафные объекты и призовые пулы. Ассеты скачиваются в кэш WebView.

---

### 6. Телеметрия и Аналитика

Для отслеживания бизнес-показателей (KPI) внедряется трекинг событий:

* Отслеживание метрик воронки: `game_loaded`, `round_started`, `round_finished`, `rare_item_cut`, `promo_dropped`, `promo_applied_in_cart`.
* Контроль производительности: трекинг FPS на разных устройствах и времени загрузки WebView для оптимизации.
