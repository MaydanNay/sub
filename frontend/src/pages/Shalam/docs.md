## Техническое задание (ТЗ): Разработка Web-игры «Shalam: Агент 007»

> **Цель проекта:** Разработка браузерного 2D-раннера (HTML5) для кросс-маркетинговой кампании Salam Premium Cola и Aviata.kz, нацеленного на сбор лидов (email/телефоны) и генерацию трафика.
> **Форм-фактор:** Адаптивное Web-приложение (Mobile-first, работает в любом браузере без скачивания).

### 1. Архитектура и Технологический стек

Игра должна быть легковесной для быстрого старта даже на слабом мобильном интернете, а бэкенд должен выдерживать резкие скачки трафика (DDoS-подобные нагрузки) после запуска Pre-roll рекламы на YouTube и сканирования QR-кодов с банок.

| Компонент | Технологии / Требования |
| --- | --- |
| **Frontend (Игровой движок)** | Phaser.js или PixiJS (HTML5 Canvas / WebGL). Отрисовка векторной (SVG) и растровой графики. |
| **Анимация и Звук** | Spine или DragonBones (для скелетной анимации персонажа и боссов). Howler.js (для управления звуками и музыкой без задержек). |
| **Backend (Core API)** | Node.js + Express. REST API для обработки результатов, выдачи промокодов и регистрации пользователей. |
| **Базы данных** | PostgreSQL (основное хранилище). Redis (для In-Memory кэширования лидербордов Топ-100 в реальном времени). |
| **Интеграции и Рассылки** | SendGrid / Mailgun (автоматическая отправка писем с промокодами Aviata). Open Graph API (для генерации красивых preview-карточек при шеринге в соцсети). |

### 2. Структура Базы Данных (PostgreSQL)

Бэкенд-команде необходимо реализовать следующую реляционную структуру для обеспечения консистентности данных розыгрыша:

* **Таблица `players` (Пользователи):**
* `id` (UUID, Primary Key)
* `contact_info` (Email или Телефон — уникальный индекс)
* `best_time` (Лучшее время прохождения в миллисекундах)
* `levels_completed` (Int, от 0 до 3)
* `referral_code` (Уникальный сгенерированный код игрока)
* `friends_invited` (Счетчик рефералов)
* `raffle_entries` (Количество билетов/шансов для финального розыгрыша)


* **Таблица `results` (Сессии/Попытки):**
* `attempt_id` (UUID)
* `player_id` (Foreign Key -> players.id)
* `level` (ID уровня)
* `survival_time` (Время жизни на уровне)
* `status` (Enum: 'win', 'lose', 'abandoned')


* **Таблица `promocodes` (Склад призов):**
* `code` (Строка, Primary Key)
* `type` (Enum: 'aviata_5000', 'aviata_10000', 'cola_free')
* `player_id` (Foreign Key, кому выдан, nullable)
* `expires_at` (Timestamp)

### 3. Игровая логика и Механика (Frontend Core)

Разработчикам клиента нужно реализовать классический паттерн Endless Runner с конечными состояниями (уровнями по 90 секунд).

* **Управление (Кроссплатформенность):** * *Прыжок:* Одинарный тап (смартфон) / Пробел или Стрелка Вверх (ПК).
* *Двойной прыжок:* Двойной тап / Двойное нажатие (открывается при получении бустера).
* *Стрельба («Шипучая пушка»):* Зажатие экрана / Клик или зажатие ЛКМ.


* **Состояния главного героя:** Базовое состояние (3 хитпоинта / Сердца). Режим неуязвимости (смена tint/свечения спрайта и игнорирование коллизий с препятствиями на 5-10 секунд). Состояние полета (джетпак с изменением гравитации по оси Y).
* **Генерация уровня:** Спавн препятствий, врагов и баффов должен происходить за пределами видимости экрана (culling) на основе заранее прописанных паттернов или сидов (чтобы избежать непроходимых ситуаций).
* **Босс-файты:** По истечении 90 секунд генерация обычных препятствий останавливается. Камера фиксируется, запускается скрипт поведения Босса (например, спавн вихрей от «Мистера Ветра»), у которого есть своя полоса здоровья, уменьшающаяся от попаданий крышками.

### 4. Безопасность, Античит и Риски

Учитывая ценные призы (поездка в Дубай, сертификаты до 100 000 тг), игра станет мишенью для скриптов и накруток.

* **Серверная валидация результатов:** Клиент не должен отправлять на сервер просто пакет `{ score: 9999 }`. Необходимо отправлять хэшированный лог ключевых действий (timestamps прыжков, подбора бонусов) с подписью (солью), чтобы бэкенд мог перепроверить, что 90-секундный уровень не был пройден за 2 секунды.
* **Rate Limiting:** Ограничение количества попыток старта игры/запросов к API с одного IP-адреса для предотвращения брутфорса лидерборда.
* **Отказоустойчивость:** Использование облачного авто-скейлинга на время старта промо-кампании. При падении БД или редиса, Frontend должен уметь кэшировать результат в `localStorage` браузера и отправлять его при восстановлении связи, выводя игроку заглушку "Ваш результат сохраняется...".

### 5. Требования к Аналитике и Виральности

* **Динамический Open Graph:** Сервер должен на лету генерировать картинку `.png` для тегов `<meta property="og:image">`, куда подставляется имя игрока и его время (например: *«Я освободил Астану за 78 секунд!»*), чтобы при отправке ссылки в WhatsApp/Telegram отображалось красивое персонализированное превью.
* **Трекинг событий:** Настроить отправку кастомных ивентов в Google Analytics (или Yandex.Metrica): `game_start`, `level_1_complete`, `boss_fight_start`, `promocode_claimed`, `click_to_aviata`.
