## Техническое задание (ТЗ): Разработка платформы ShuQaz

> 
> **Цель проекта:** Создание геймифицированного мобильного приложения, Telegram Mini App и PWA для изучения казахского языка, основанного на механике заботы о виртуальном питомце (Снежном Барсе по имени Барыс).
> **Бизнес-задача:** Повысить удержание пользователей (Retention) с 15% до 60% и DAU до 45%.
> 
> 

### 1. Архитектура и Технологический стек

Проект требует создания гибкой экосистемы, доступной на разных платформах, с единой базой данных и синхронизацией прогресса.

| Компонент | Технологии / Требования |
| --- | --- |
| **Frontend (Клиентские приложения)** | React Native или Flutter для кроссплатформенной разработки (iOS/Android). PWA для Web-версии. Отдельная адаптация интерфейса для Telegram Mini App.

 |
| **Анимация и Визуал** | Использование Lottie или Spine для 2D-анимации персонажа (Барса). Плоский дизайн (flat) с национальными орнаментами.

 |
| **Backend (Core API)** | Node.js + Express или Nest.js. Микросервисная архитектура для разделения логики словаря, магазина и контента.

 |
| **Базы данных** | PostgreSQL (для хранения профилей пользователей, их личных словарей и прогресса).

 |
| **Кэширование и Real-time** | Redis для кэширования лидербордов и данных в реальном времени.

 |
| **Хранилище медиа** | S3-совместимое хранилище для медиаконтента раздела "Жайляу" (аудио, видео, подкасты).

 |

### 2. Структура Базы Данных (PostgreSQL)

Бэкенд-команде необходимо реализовать реляционную структуру для связывания контента, словаря и геймификации:

* 
**`users`:** Хранит базовые данные, баланс валюты (баурсаки), уровень опыта (XP), текущий Streak (серию дней), статус подписки (Free / Premium).


* 
**`dictionary_words`:** Личный словарь пользователя ("Сандық"). Содержит связи ID пользователя и ID слова, а также текущий статус освоения: Жаңа (Новое), Үйренуде (Учу), Білем (Знаю), Мастер.


* 
**`content_library`:** Таблица библиотеки "Жайляу". Хранит метаданные о песнях, видео, текстах, их уровень сложности (А1-С1) и жанр.


* 
**`inventory_cosmetics`:** Хранит купленные пользователем элементы кастомизации (например, Тюбетейка, Чапан, Кепка).

### 3. Интеграции и Внешние API

* 
**API Словарей:** Интеграция с парсерами или официальными API Sozdik.kz / Tilalemi.kz для автоматического подтягивания перевода, контекста и транскрипции при клике на слово в тексте.


* 
**Audio & TTS:** Генерация произношения слов через Text-to-Speech (TTS) или использование заранее записанной базы от носителей языка.


* 
**Платежные шлюзы:** Интеграция Kaspi и CloudPayments для обработки транзакций при покупке Premium-подписки (299-499 тенге/месяц) и отдельных курсов (от 1490 до 2990 тенге).

### 4. Игровая и Образовательная логика (Core Loop)

Разработчикам необходимо внедрить алгоритмы интервального повторения и экономику внутриигровой валюты.

* 
**Алгоритм Spaced Repetition (Интервальное повторение):** Система должна анализировать ошибки в 6 типах мини-игр (Сөз-Конструктор, Аудио-Вызов, Аудармашы, Көкпар, Сөйлем құру, Диктант) и чаще предлагать слова, которые пользователь забывает.


* **Экономика "Баурсаков":** Начисление soft currency за действия на сервере (во избежание накруток). Например: +2 за новое слово, +10 за тренировку, +15 за урок грамматики.


* 
**Логика Питомца (Барса):** * Шкала сытости имеет лимит (цель: 100 баурсаков в день) и обнуляется каждые 24 часа.


* Уровневая система: опыт (XP) равен скормленным баурсакам (1 баурсак = 1 XP). При накоплении XP меняется стадия Барса (от Балапан до Ата).


* Барс не умирает, но меняет анимацию на "вялую", если нет входа 2 дня.




* **Streak Mechanism (Серия дней):** Ежедневный скрипт проверяет, покормил ли пользователь Барса хотя бы на 50%. Если да — streak +1. Если нет — обнуление (если не применен предмет "Заморозка").

### 5. Телеметрия и Аналитика KPI

Для оценки эффективности MVP необходимо заложить трекинг следующих продуктовых и образовательных метрик:

* 
**Бизнес-метрики:** Трекинг DAU (цель 45%), Retention 7-го дня (цель 50%), Retention 30-го дня (цель 30%) и конверсии в Premium (цель 8-12%).


* 
**Образовательные метрики:** Среднее количество выученных слов (статус "Мастер"), процент пользователей, достигших уровня А2 и В1.
